## How I Think

**What happens when 1,000,000 users hit the system at once?**  
→ I simulate it first. Load-test at 2× expected peak, introduce queues, scale horizontally.

**What if a database node fails mid-transaction?**  
→ Replica failover + idempotent operations. Failure becomes invisible.

**What if AI inference becomes a bottleneck?**  
→ Async queues + caching + model warm pools. No user-facing delays.

**What if the API gets flooded with 50,000 req/sec?**  
→ Edge protection drops malicious traffic before it reaches the app.

**What if traffic spikes instantly?**  
→ Event-driven queues absorb it. Systems scale automatically.

**What if a memory leak appears?**  
→ Observability catches it early. Containers self-heal.
